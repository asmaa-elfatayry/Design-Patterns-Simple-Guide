Command Pattern is a **behavioral design pattern** that **encapsulates a request as an object**, allowing you to **decouple the sender (requester) from the receiver (executor)**.

It enables you to pass commands around, queue them, log them, or support undoable operations â€” all without the sender knowing the details of the execution.

* Command encapsulates an **action as an object**.

* Facilitates **undo/redo and queueing** of operations.
* Promotes **composition over inheritance**.
